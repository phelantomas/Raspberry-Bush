Recipe

Now that we have gathered our ingredients, we now need to prepare them. This requires firstly assembling the ClusterHat, and then flashing the images to your micro SD cards. The ClusterHat site itself has a pretty robust guide on this, so we will link to it here. We recommend using etcher to flash the sd cards.

Once assembled, flashed, and booted, follow these steps:
  1. To begin with, if you haven't already, turn on the clusterhat with the command, “clusterhat on”.
  2. Turn ssh on by default, it will initially be off. There are two ways to do this, by using the gui, or the terminal.
    1. The gui way.
    2. The terminal way. Type in “sudo raspi-config”. Go into interfaces, then select ssh, and finally, select yes. It will now be turned on by default.
  3. Once done on the main one, needs to be done to the rest of the pi zeros. Do this, you need to firstly need to use the command “minicom” follow by the hostname of the pi zero, for example, “minicom p1”. Once in, follow step 2b to setup ssh. Repeat for all pi zeros.
  4. Once done, generate an rsa key, “ssh-keygen -t rsa”.
  5. To get passwordless access into the pi’s, run the following command, “for I in 1 2 3 4 sshit”.
  6. If you are using v1.3 of pi zeros, you will run into the issue of having no wifi.
    1. ssh into each of the pi zeros and add your wifi details wpa_supplicant.conf file. This will make more sense in a bit when we are installing docker. Just “sudo nano /etc/wpa_supplicant/wpa_supplicants.conf” and add your network details i.e.
      network = {
      ssid="HOMEWIFI"
      psk="12345678"
      }
    2. You may have to reboot, to do so, “sudo reboot”.
  7. Once all done, we can now move onto actually installing docker. Run the following command to download docker, “curl -sSL https://get.docker.com | sh”.
  8. Once downloaded, run “sudo usermod -aG docker pi”. Now 'pi' user can run docker without needing sudo.
  9. Run “docker version” to make sure everything installed correctly.
  10. The same now needs to be done for all the pi zeros. However you cannot just ssh into them as they have no connection to the internet. If you have wifi enabled pi zeros you can skip this step and simply install using ssh.
    1. Turn off your machine, remove the sd card from the main pi 3 b and put in the sd card from p1.
    2. Boot up the machine. You will not be able to do anything else now as it has no drivers for mouse or keyboard.
    3. Boot up the machine. You will not be able to do anything else now as it has no drivers for mouse or keyboard.
    4. You will need a seperate computer to ssh into, we used windows, and putty to do so.
    5. To ssh into the device, there are multiple ways, however, only one worked for us. On boot up, it displays the ip address on the screen. Use this to gain access. Once in, simply follow step 6.
    6. Do the same for the rest of the sd cards, the ip address should be same for them all.
  11. Once docker is installed on all pi’s, run “docker info|grep Swarm”. You should see that the swarm is inactive.
  12. To start a swarm, run the following command on the main controller “docker swarm init”, or “docker swarm init --advertise-addr <ip>”.
  13. A token should be generated, “docker swarm join \
    --token <random token docker generated> \
    <controller ip>:”.
    This needs to be pasted into each of the pi zeros, however there is a slight problem. The devices have no real time clocks, and are years behind, if you run this command it will simply state that the certificate is invalid. To fix this you will need to:
    1. Change the time to the current using the command “sudo date -s "11 MAR 2018 11:11:11".
    2. To the same for all the pi zeros. It doesn't matter if there a couple minutes off.
  14. Once the times have been set, paste the token in each of the other pi zeros to add them to the swarm.
  15. To check that all have joined, run “docker node ls”.
You have successfully got a docker swarm running on your clusterhat, well done!
